## 幻灯片 3：研究背景

**自动呼吸声分类（ARSC）的临床价值**

- 无创诊断肺部疾病（哮喘、肺炎、COPD）
- 远程医疗和资源受限地区的早期筛查
- 数字听诊器 + AI → 客观化、标准化诊断

| 参与者 | 126 人 |
| 采样率 | 4kHz - 44.1kHz（重采样至 16kHz） |
| 划分 | 60% 训练 / 40% 测试（患者不重叠） |
| 类别 | normal (~53%), crackle (~25%), wheeze (~14%), both (~7%) |

## 幻灯片 4：面临的挑战

**三大核心挑战**

| 挑战 | 详细说明 |
|------|---------|
| **噪声污染** | 肢体运动伪影、心音干扰、环境噪声、传感器伪影 |
| **数据稀缺** | 带注释样本不足，类别严重不平衡 |
| **传统去噪缺陷** | 巴特沃斯滤波丢失高频裂音（>2000Hz），小波分解丢失瞬态特征 |

**传统"显式去噪→分类"范式的问题**

```
传统流程：原始音频 → 显式去噪 → 分类
                    ↑
              问题出在这里！
              1. 通用去噪器不适用于呼吸声
              2. 去噪过程中丢失病理特征
              3. 去噪和分类目标解耦
```

## 幻灯片 5：核心贡献

**ADD4RSC：隐式去噪，端到端联合优化**

**三大创新**

1. **自适应频率滤波器（AFF）**
   - 可学习频谱掩码 + 软收缩
   - 消除噪声同时保留诊断高频成分

2. **差分去噪层（DDL）**
   - 多头差分注意力（MHDA）
   - 通过增强样本比较降低噪声变化

3. **偏置去噪损失（BDL）**
   - 联合优化分类和鲁棒性
   - 无需干净标签

**核心结果**：ICBHI 2017 达到 **65.53% Score**，比前 SOTA 提升 **1.99%**

## 幻灯片 7：自适应频率滤波器（AFF）

**频域处理机制**

**数学公式**：

$$\hat{X}(t,f) = F^{-1}[S_\alpha(M(X(U,V))) \odot X(U,V)]$$

**处理流程**：

```
Mel频谱图
  ↓ FFT（时域 → 频域）
  ↓
可学习掩码 M(·)：Linear → ReLU → Linear
  ↓
软收缩 S_α(·)：sign(x)·max(|x|-α, 0)，α=0.02
  ↓
掩码 × 频谱（逐元素相乘）
  ↓ iFFT（频域 → 时域）
滤波后的频谱图
```

**关键优势**：
- 每个输入样本生成独特的频谱掩码（实例自适应）
- 软收缩控制稀疏度，避免硬截断导致的信息丢失
- 保留 >2000Hz 的高频裂音特征

## 幻灯片 8：差分去噪层（DDL）

**多头差分注意力（MHDA）**

**核心公式**：

$$A = \left(\sigma\left(\frac{Q_1K_1^T}{\sqrt{d}}\right) - \lambda \cdot \sigma\left(\frac{Q_2K_2^T}{\sqrt{d}}\right)\right)V$$

**工作原理**：

```
输入 X
  ↓ 线性投影
Q, K, V（各 2·d_head·h 维）
  ↓ chunk 分割
Q₁, Q₂  |  K₁, K₂  |  V
  ↓              ↓
路径1: A₁=softmax(Q₁K₁ᵀ/√d)
路径2: A₂=softmax(Q₂K₂ᵀ/√d)
  ↓              ↓
差分: Attention = A₁ - λ·A₂
  ↓
应用: Output = Attention · V
```

**λ 可学习缩放因子**：
- λ = exp(λ_q1·λ_k1) - exp(λ_q2·λ_k2) + λ_init
- 4 个可学习参数控制两路注意力的权重平衡

**与传统注意力对比**：

| 传统注意力 | 差分注意力 (MHDA) |
|-----------|------------------|
| softmax(QKᵀ)·V | (softmax(Q₁K₁ᵀ) - λ·softmax(Q₂K₂ᵀ))·V |
| 单路 softmax | **双路差分** |
| 对噪声敏感 | **过滤噪声敏感变化** |

## 幻灯片 9：偏置去噪损失（BDL）

**混合损失设计**

$$L = \beta \cdot L_{\text{Bias Denoise}} + (1-\beta) \cdot L_{\text{CE}}$$

**偏置去噪损失**（标签平滑交叉熵）：

$$L_{\text{Bias Denoise}} = -\sum_i \left[y_c(1-\epsilon) + \frac{\epsilon}{C}\right] \log[\phi(\text{Norm}(p))]$$

**参数设置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| β | 0.5 | 去噪与分类权重平衡 |
| ε | 0.2 | 不确定性缓冲器 |
| C | 4 | 分类总数 |
| φ | 1×1 Conv | 卷积层 |

**设计优势**：
- 仅依赖分类标签，无需干净参考信号
- 标签平滑防止模型过度自信
- 联合优化分类准确性和去噪鲁棒性

## 幻灯片 14：本地复现 — 第二次训练（优化）

**关键改进**

```bash
python main.py --imagenet_pretrained --weighted_sampler --weighted_loss \
    --batch_size 16 --eval_freq 2 --early_stop_patience 4
```

| 改进项 | 说明 | 影响 |
|--------|------|------|
| ImageNet 预训练 | 加载预训练权重 | 加速收敛 |
| 加权采样器 | 平衡各类样本出现频率 | **Se 大幅提升** |
| 加权损失 | 增加少数类损失权重 | **Se 大幅提升** |
| Batch Size 16 | 加倍 batch | 训练速度翻倍 |
| 早停 | patience=4 | 防止过拟合 |

**训练耗时**：约 17 小时

**结果**：

| 指标 | 复现结果 | 论文结果 | 差距 |
|------|---------|---------|------|
| Sp | 68.89% | 83.76% | -14.87% |
| Se | **54.22%** | 34.18% | **+20.04%** |
| Score | **61.55%** | 58.97% | **+2.58% ✅** |

**Score 61.55% 超过论文 58.97%！**

## 幻灯片 15：复现结果对比总结

| 训练 | Sp(%) | Se(%) | Score(%) | 耗时 | 关键改动 |
|------|-------|-------|----------|------|---------|
| 第一次（默认） | 89.94 | 7.42 | 48.68 | 2.5天 | 无 |

- 加权采样 + 加权损失 → Se 从 7% 跃升至 54%
- 优化后 Score 超过论文，但 Sp/Se 分布不同
- 我们的模型更擅长检测异常（高 Se），论文模型更擅长识别正常（高 Sp）

## 幻灯片 16：过拟合分析

**第二次训练过拟合诊断**

| 指标 | 值 |
|------|-----|
| 训练准确率（最终） | 89.50% |
| 测试准确率（最终） | 60.86% |
| **差距** | **28.64%**（严重过拟合 >25%） |
| 最佳 Score Epoch | 36 |
| 最大差距 Epoch | 40（29.35%） |

**训练曲线特征**：
- 训练准确率持续上升（27% → 89.5%）
- 测试准确率在 Epoch 36 后停滞甚至下降
- Score 峰值出现在 Epoch 36

**改进建议**：
1. Early Stopping（Epoch 36 停止）
2. 增加 Dropout (0.3-0.5)
3. 增大 Weight Decay (1e-4 ~ 5e-4)
4. 数据增强（SpecAugment 加强、时移、添加噪声）

## 幻灯片 18：复现经验总结

**复现差距根因分析**

| 因素 | 影响程度 | 说明 |
|------|---------|------|
| 类别不平衡处理 | ⭐⭐⭐ 高 | weighted_sampler + weighted_loss 是关键 |
| 数据划分协议 | ⭐⭐⭐ 高 | 随机划分 vs official split |
| 预训练权重 | ⭐⭐ 中 | ImageNet/AudioSet 预训练加速收敛 |
| ADD 模块实现 | ⭐⭐ 中 | denoise_depth 实际控制 lambda_init |
| 训练细节 | ⭐ 低 | loss 口径差异、AMP GradScaler |

**复现建议**：
1. 务必启用 `--weighted_sampler --weighted_loss`
2. 使用预训练权重（ImageNet for ResNet, AudioSet for AST）
3. 设置 Early Stopping 防止过拟合
4. 增大 Batch Size 加速训练
