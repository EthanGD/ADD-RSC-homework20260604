## 一、研究背景

### 1.1 自动呼吸声分类（ARSC）的重要性

自动呼吸声分类（Automated Respiratory Sound Classification）是医学人工智能领域的重要研究方向，其核心价值在于：

- **无创诊断肺部疾病**：通过分析数字听诊器或可穿戴传感器采集的听诊信号，辅助诊断哮喘、肺炎、慢性阻塞性肺疾病（COPD）等呼吸系统疾病。
- **远程医疗**：在医疗资源匮乏的地区实现早期筛查，降低对专业医生的依赖。
- **客观化诊断**：传统听诊依赖医生主观经验，自动化分类可提供标准化、可重复的诊断依据。

### 1.2 面临的核心挑战

| 挑战 | 说明 |
|------|------|
| **数据稀缺** | 带注释的呼吸声样本不足，且标注成本高 |
| **噪声污染** | recordings 中混杂肢体运动伪影、心音干扰、环境噪声、传感器伪影等 |
| **传统去噪局限** | 谱减法、维纳滤波等传统方法无法有效保留高频病理特征（如裂音 >2000Hz） |
| **类别不平衡** | ICBHI 数据集中正常样本约占 53%，wheeze 仅占 14%，both 仅占 7% |

### 1.3 现有方法的局限性

传统"顺序去噪→分类"范式存在三大根本问题：

1. **区域不匹配**：通用音频去噪器在非生物医学记录上训练，不适用于呼吸声场景
2. **病理特征丢失**：显式去噪模块无意中消除细微但关键的病理特征
3. **目标解耦**：级联结构阻碍了噪声不变表示的联合优化

传统滤波方法的具体缺陷：
- **巴特沃斯带通滤波**：过度衰减 100-2000Hz 以外频率，导致高频裂音（>2000Hz）信息丢失
- **小波分解**：依赖固定基函数，导致瞬态特征（如 crackle 的短时脉冲）丢失

## 二、核心贡献

本文提出 **ADD4RSC**（Adaptive Differential Denoising for Respiratory Sounds Classification），一种**隐式去噪**方法，将噪声抑制与病理特征保留统一在端到端框架中。三大创新如下：

### 2.1 自适应频率滤波器（Adaptive Frequency Filter, AFF）

- **核心思想**：在频域中通过可学习的频谱掩码自动识别并抑制噪声频率，同时保留对诊断有价值的高频成分
- **技术实现**：
  - FFT 将 Mel 频谱图转换到频域
  - 可学习实例自适应掩码 M(·)：线性层 → ReLU → 线性层，生成与输入相关的频谱掩码
  - 软收缩 S_α(·)：S_α(x) = sign(x)·max{|x|-α, 0}，控制频谱稀疏度（α=0.02）
  - iFFT 将滤波后的频域信号还原回时域
- **优势**：相比固定滤波器，能根据每个输入样本自适应调整滤波策略

### 2.2 差分去噪层（Differential Denoise Layer, DDL）

- **核心思想**：利用多头差分注意力（Multi-Head Differential Attention, MHDA）机制，通过两路注意力的差分来抑制噪声扰动
- **技术实现**：
  - 将 Q、K 各分为两组：Q₁/Q₂ 和 K₁/K₂
  - 计算两路独立注意力：A₁ = softmax(Q₁K₁ᵀ/√d)，A₂ = softmax(Q₂K₂ᵀ/√d)
  - 差分注意力：Attention = A₁ - λ·A₂，其中 λ 为可学习缩放因子
  - λ = exp(λ_q1·λ_k1) - exp(λ_q2·λ_k2) + λ_init
- **优势**：过滤噪声敏感变化的同时，保留稳定的高置信度特征

### 2.3 偏置去噪损失（Bias Denoising Loss, BDL）

- **核心思想**：联合优化分类准确性和去噪鲁棒性，无需干净参考信号
- **技术实现**：
  - 混合损失：L = β·L_BiasDenoise + (1-β)·L_CE
  - 偏置去噪损失采用标签平滑交叉熵：L_BiasDenoise = -Σ[y_c(1-ε) + ε/C]·log[φ(Norm(p))]
  - 参数：β=0.5（权重平衡），ε=0.2（不确定性缓冲器）
- **优势**：仅依赖分类标签，无需配对的干净/噪声数据

## 三、模型架构

### 3.1 整体流程

```
输入音频
  ↓
预处理（重采样16kHz → 固定8秒 → 幅度归一化 → Mel频谱图）
  ↓
自适应频率滤波器（AFF）—— 频域可学习掩码 + 软收缩
  ↓
差分去噪层（DDL）—— 多头差分注意力（MHDA）
  ↓
Backbone（ResNet50 或 AST）—— 特征提取
  ↓
分类器 —— 4类输出（normal / crackle / wheeze / both）
```

### 3.2 关键模块封装

三大核心模块封装在 `DiffTransformerLayer` 类中：

| 模块 | 代码实现 | 功能 |
|------|---------|------|
| **AFF** | `AFNO1D` (自适应频域滤波器) | 频域可学习掩码滤波 |
| **DDL** | `MultiHeadDifferentialAttention` | 多头差分注意力去噪 |
| **BDL** | `LabelSmoothingLoss` | 标签平滑偏置去噪损失 |

### 3.3 两种骨干网络

| 特性 | ResNet50 | AST (Audio Spectrogram Transformer) |
|------|---------|-------------------------------------|
| 架构类型 | CNN（卷积神经网络） | Transformer（自注意力机制） |
| 预训练方式 | 随机初始化 / ImageNet预训练 | AudioSet 预训练权重微调 |
| 输入处理 | Mel频谱图 → 卷积提取局部特征 | Mel频谱图 → 切patch → 全局注意力建模 |
| 参数量 | 较小 | 较大（需加载 ~340MB 预训练权重） |
| 论文结果 | Score 58.97% | Score 65.53%（SOTA） |

**AST（Audio Spectrogram Transformer）** 是 MIT 于 2021 年提出的首个无卷积纯注意力音频分类模型，将音频频谱图视为"图像"，通过 ViT 架构处理。其优势包括：
- 完全无卷积，从底层捕捉全局信息
- 可直接迁移 ImageNet 预训练权重
- 在 AudioSet、ESC-50 等数据集上达到 SOTA

## 四、实验设置与结果

### 4.1 数据集：ICBHI 2017

| 属性 | 值 |
|------|-----|
| 参与者 | 126 人 |
| 采样率 | 4kHz - 44.1kHz（重采样至 16kHz） |
| 持续时间 | 10-90 秒/记录 |
| 划分方式 | 60% 训练 / 40% 测试（患者不重叠） |
| 总呼吸周期 | ~6898 个（训练 4213 + 测试 2685） |
| 类别分布 | normal ~53%, crackle ~25%, wheeze ~14%, both ~7% |

### 4.2 评估指标

- **特异性 Sp** = 正确识别的正常声音数 / 总正常声音数
- **敏感性 Se** = 正确识别的异常声音数 / 总异常声音数
- **综合 Score** = (Sp + Se) / 2

### 4.3 论文 SOTA 结果

**CNN 方法对比：**

| CNN 模型 | Sp(%) | Se(%) | Score(%) | 发表 |
|---------|-------|-------|----------|------|
| LungRN+NL | 63.20 | 41.32 | 52.26 | BioCAS 2019 |
| RespireNet | 72.30 | 40.10 | 56.20 | EMBC 2021 |
| CoTuning | 79.34 | 37.24 | 58.29 | IEEE TBE |
| SCL | 75.95 | 39.15 | 57.55 | WASPAA 2023 |
| **ADD4RSC (ResNet50)** | **83.76** | 34.18 | **58.97** | **Interspeech 2025** |

**Transformer 方法对比：**

| Transformer 模型 | Sp(%) | Se(%) | Score(%) | 发表 |
|-----------------|-------|-------|----------|------|
| AST Fine-tuning | 77.14 | 41.97 | 59.55 | Interspeech 2023 |
| Patch-Mix CL | 81.66 | 43.07 | 62.37 | Interspeech 2023 |
| M2D | 81.51 | 45.08 | 63.29 | IEEE TASLP |
| BTS | 81.40 | 45.67 | 63.54 | Interspeech 2024 |
| **ADD4RSC (AST)** | **85.13** | **45.94** | **65.53** | **Interspeech 2025** |

### 4.4 消融实验

| 模型变体 | Sp(%) | Se(%) | Score(%) |
|---------|-------|-------|----------|
| w/ AFF only | 82.47 | 44.47 | 63.47 |
| w/ DDL only | 83.73 | 44.03 | 63.88 |

## 五、本地复现记录

### 5.1 环境配置

- **硬件**: NVIDIA L20 GPU (46GB VRAM)
- **框架**: PyTorch 2.0.1 + CUDA 11.7
- **服务器**: 8×L20 GPU 集群

### 5.2 第一次训练（默认参数，ResNet50）

**训练命令**:
```bash
python main.py --data_folder ./ICBHI/ICBHI_final_database
```

**训练参数**:
- 模型: ResNet50（从头训练，无预训练）
- Epochs: 50, Batch Size: 8, LR: 5e-5
- 未启用 weighted_sampler / weighted_loss
- 未使用 ImageNet 预训练

**训练耗时**: 2026-06-16 11:38 ~ 2026-06-18 17:21（约 2.5 天）

**结果**:
| 指标 | 值 | 论文值 | 差距 |
|------|-----|--------|------|
| Sp | 89.94% | 83.76% | +6.18% |
| Se | 7.42% | 34.18% | **-26.76%** |
| Score | 48.68% | 58.97% | **-10.29%** |

**问题分析**:
- Se 极低（7.42%），模型几乎把所有样本预测为"正常"
- 根本原因：类别不平衡未处理（normal 占 53%，both 仅 7%）
- 默认参数下未启用加权采样器和加权损失

### 5.3 第二次训练（优化参数，ResNet50）

**训练命令**:
```bash
python main.py --imagenet_pretrained --weighted_sampler --weighted_loss --batch_size 16 --eval_freq 2 --early_stop_patience 4
```

**关键改进**:
| 改进项 | 说明 |
|--------|------|
| `--imagenet_pretrained` | 使用 ImageNet 预训练初始化 ResNet50 |
| `--weighted_sampler` | 启用加权采样器，平衡各类样本 |
| `--weighted_loss` | 启用加权损失函数 |
| `--batch_size 16` | 加大 batch size，加速训练 |
| `--eval_freq 2` | 每 2 epoch 验证一次 |
| `--early_stop_patience 4` | 连续 4 次不提升则早停 |

**训练耗时**: 约 17 小时（每轮约 20 分钟）

**结果**:
| 指标 | 值 | 论文值 | 差距 |
|------|-----|--------|------|
| Sp | 68.89% | 83.76% | -14.87% |
| Se | **54.22%** | 34.18% | **+20.04%** |
| Score | **61.55%** | 58.97% | **+2.58%** ✅ |

**关键发现**:
- **Score 61.55% 超过论文 58.97%，提升 2.58%！**
- Se 大幅提升至 54.22%（比论文高 20%），模型更擅长检测异常呼吸声
- Sp 偏低（68.89%），存在一些正常样本被误判为异常
- 最佳结果出现在 Epoch 36

### 5.4 过拟合分析

| 指标 | 值 |
|------|-----|
| 训练准确率 | 89.50% |
| 测试准确率 | 60.86% |
| 差距 | 28.64%（严重过拟合 >25%） |

**观察**:
- 训练准确率持续上升（27% → 89.5%）
- 测试准确率在 Epoch 36 后停滞甚至下降
- 最佳 Score 出现在 Epoch 36，之后开始过拟合

### 5.5 单样本预测验证

对 `101_1b1_Al_sc_Meditron.wav` 进行预测：

| 分类 | 概率 |
|------|------|
| crackle（爆裂音） | 61.71% |
| both | 29.89% |
| wheeze | 5.41% |
| normal | 3.00% |

### 5.6 复现差距分析

| 因素 | 影响 | 说明 |
|------|------|------|
| 数据划分 | 高 | 代码使用随机划分，与论文 official split 可能不完全一致 |
| 类别不平衡处理 | 高 | 第二次训练启用后效果显著 |
| ADD 模块实现 | 中 | denoise_depth 参数实际控制 lambda_init 而非层堆叠 |
| 训练/验证 loss 口径 | 低 | 训练用 β 加权，验证未加权 |
| ImageNet 预训练 | 中 | 第二次训练启用后加速收敛 |

## 六、结论与启示

1. **ADD4RSC 方法有效**：三大创新（AFF + DDL + BDL）协同工作，在 ICBHI 2017 上达到 SOTA
2. **复现需注意细节**：默认参数无法复现论文结果，需正确配置类别平衡、预训练权重等
3. **ResNet50 潜力大**：优化参数后，ResNet50+ADD 的 Score 超过论文报告值
4. **过拟合是主要挑战**：训练-测试差距大，建议 Early Stopping + 正则化
5. **方法架构无关**：ADD 模块可同时应用于 CNN 和 Transformer，具有通用性
