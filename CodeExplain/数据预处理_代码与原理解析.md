## 1. 总体流程图

整个预处理链路可以概括为：

1. 读取 ICBHI 数据目录中的 `.wav` 和 `.txt`
2. 生成 train / test 划分
3. 根据标注把每条录音切成单个呼吸周期
4. 对每个周期做定长裁剪 / padding
5. 把波形转成 Mel filter bank（fbank）特征图
6. 训练阶段再做 `ToTensor -> SpecAugment -> Resize`
7. 最终送入模型

可以用一句话概括：

> 原始呼吸音录音先被切成“单周期样本”，再统一长度、转为频谱图，并在训练阶段做谱图增强，最后作为 2D 输入送进 ResNet50 或 AST。

## 2. DataLoader 入口：训练和验证各自做什么

在 [main.py:set_loader()](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/main.py#L169-L216) 中：

```python
args.h, args.w = 1024, 256
train_transform = [transforms.ToTensor(),
                    SpecAugment(args),
                    transforms.Resize(size=(int(args.h * args.resz), int(args.w * args.resz)))]
val_transform = [transforms.ToTensor(),
                transforms.Resize(size=(int(args.h * args.resz), int(args.w * args.resz)))]
```

### 训练集

训练集预处理顺序：

1. `ToTensor()`
2. `SpecAugment(args)`
3. `Resize((1024, 256))`

### 验证集

验证集预处理顺序：

1. `ToTensor()`
2. `Resize((1024, 256))`

也就是说：

- **训练集有谱图增强**
- **验证集没有增强**

这是标准做法，因为验证阶段希望尽量反映“真实模型表现”，而不是“增强后的训练样本表现”。

## 6. 波形预处理：重采样 + 淡入淡出 + 切周期

在 [get_individual_cycles_torchaudio()](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/util/icbhi_util.py#L131-L169) 中：

```python
data, _ = librosa.load(fpath, sr=sample_rate)
data = torch.from_numpy(data).unsqueeze(0).float()
```

这里 `librosa.load(..., sr=sample_rate)` 的作用是：

- 统一把音频采样率重采样为 `args.sample_rate`
- 默认值是 `16000`

### 为什么统一到 16kHz

不同录音设备可能采样率不同，统一采样率有利于：

- 保证特征维度稳定
- 让不同录音的频谱可比较

随后代码又加了淡入淡出：

```python
fade = T.Fade(fade_in_len=fade_samples, fade_out_len=fade_samples, fade_shape='linear')
data = fade(data)
```

作用是：

- 减少音频开头和结尾的突变
- 降低切段后产生的边界不连续
- 让频谱图更平滑，减少人为边界噪声

## 7. 定长处理：为什么每个呼吸周期都要统一成 8 秒

呼吸周期长短天然不一致，而 CNN / Transformer 通常更喜欢固定尺寸输入。  
所以在 [cut_pad_sample_torchaudio()](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/util/icbhi_util.py#L109-L129) 中，所有周期都会被统一到：

```python
target_duration = args.desired_length * args.sample_rate
```

默认：

- `desired_length = 8`
- `sample_rate = 16000`

所以目标长度是：

- `8 * 16000 = 128000` 个采样点

### 如果样本太长

```python
if data.shape[-1] > target_duration:
    data = data[..., :target_duration]
```

直接裁掉后半段。

### 如果样本太短

当前默认 `pad_types='repeat'`：

```python
ratio = math.ceil(target_duration / data.shape[-1])
data = data.repeat(1, ratio)
data = data[..., :target_duration]
data = fade_out(data)
```

含义是：

- 把原始呼吸周期重复拼接
- 直到够 8 秒
- 再裁成精确长度

### 为什么默认用 repeat 而不是 zero-padding

相对 zero-padding，repeat 的直觉是：

- 不会在大量位置引入纯静音
- 能保留原始周期内部的模式信息
- 对频谱图来说通常比大量零填充更自然

但它也有代价：

- 会人为重复同一段周期，带来伪周期性

这是一种工程上的折中。

## 8. 可选原始波形增强：先增强音频，再转频谱

在 [ICBHIDataset.__init__()](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/util/icbhi_dataset.py#L110-L126) 中：

这是“**先增强波形，再重算谱图**”的思路。

- 原始波形增强作用在**音频信号层面**

- `raw_augment = 0`

## 9. 从波形到频谱图：为什么用 fbank

波形最终会被转换为 fbank 特征，代码在 [generate_fbank()](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/util/icbhi_util.py#L172-L182)：

```python
fbank = torchaudio.compliance.kaldi.fbank(
    audio,
    htk_compat=True,
    sample_frequency=sample_rate,
    use_energy=False,
    window_type='hanning',
    num_mel_bins=n_mels,
    dither=0.0,
    frame_shift=10
)
```

关键参数：

- `sample_rate = 16000`
- `num_mel_bins = 128`
- `frame_shift = 10ms`

### 为什么用 fbank

相比直接喂原始波形，fbank 有几个优点：

- 把时域信号变成时频表示，更利于 CNN / ViT 处理
- 更接近人耳感知频率分布（Mel 频率）
- 医学音频里的 crackle / wheeze 往往在时频图上更容易显现模式

### 为什么这里用 kaldi 风格 fbank

因为：

- 兼容语音/音频建模常见流程
- AST 模型通常也适配这类输入表示

## 10. 特征归一化：为什么减均值除标准差

在 `generate_fbank()` 中：

```python
mean, std = -4.2677393, 4.5689974
fbank = (fbank - mean) / (std * 2)
```

这一步是固定统计量归一化，目的包括：

- 把不同样本的特征幅值拉到更稳定范围
- 降低训练初期数值波动
- 让优化器更容易收敛

注意这里不是在线计算当前 batch 的 mean/std，而是直接使用预设常数。

最后再变成：

```python
fbank = fbank.unsqueeze(-1).numpy()
```

输出形状大致是：

- `[time, freq, channel]`

其中 `channel=1`。

## 11. SpecAugment：在谱图上做人造遮挡

训练时在 [main.py](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/main.py#L184-L192) 里，`ToTensor()` 后会进入 [SpecAugment](file:///c:/Ethan/code/cdssHomeWork20260604/ADD-RSC-homework20260604/util/augmentation.py#L84-L252)。

当前默认策略是：

```python
specaug_policy = 'icbhi_ast_sup'
```

它对应：

```python
self.W, self.F, self.m_F, self.T, self.p, self.m_T = 0, 48, 2, 160, 1.0, 2
```

表示：

- 不做 time warp（`W=0`）
- 做 2 次频率遮挡（最多 48 个 mel bin）
- 做 2 次时间遮挡（最多 160 个 time frame）

### 频率遮挡

```python
self.mel_spectrogram[:, f0:f0 + f, :] = mask_value
```

随机把一段频率带盖住。

### 时间遮挡

```python
self.mel_spectrogram[:, :, t0:t0 + t] = mask_value
```

随机把一段时间区域盖住。

### 这样做的原理

本质是：

- 强迫模型不要过分依赖某一小块局部模式
- 提升鲁棒性
- 模拟真实环境中的局部缺失、噪声覆盖、设备差异

对呼吸音来说，这能减少模型对某些“偶然性的局部纹理”的过拟合。

## 12. ToTensor 和 Resize：为什么最后都统一成 `(1024, 256)`

数据进入 transform 后：

```python
transforms.ToTensor()
transforms.Resize(size=(1024, 256))
```

这里会把谱图变成张量，并统一成固定大小。

### 为什么要 Resize

因为经过切分和 fbank 后，原始谱图尺寸可能受样本长度等因素影响。  
固定到 `(1024, 256)` 有利于：

- ResNet50 作为 2D CNN 直接处理
- AST 模型使用固定 patch 网格
- batch 内样本尺寸一致，方便并行训练

这里的尺寸实际上由：

- `args.h = 1024`
- `args.w = 256`

硬编码指定。

## 14. 整个预处理流程的“设计哲学”

这套预处理背后其实有很清晰的思路：

### 14.1 以“呼吸周期”为样本单位

不是整条录音直接分类，而是：

- 先切成单个呼吸周期

这样更符合病理事件发生的粒度。

### 14.2 统一输入长度

通过裁剪 / repeat padding，把不同长度周期变成固定长度，保证模型结构简单可用。

### 14.3 使用频谱图而非原始波形

因为呼吸音异常模式（如 crackle / wheeze）在时频域中更容易表达。

### 14.4 在谱图层面增强

SpecAugment 的思想是：

- 在不破坏整体语义的前提下，增加训练扰动

提升泛化能力。

## 15. 当前实现里几个值得注意的地方

### 15.1 日志里的 “official” 不代表真的 official split

虽然代码参数名是 `test_fold='official'`，但目前实现仍然是随机 60/40。

### 15.2 频谱尺寸是手工固定到 `(1024,256)`

这不是由真实 fbank 输出动态决定，而是：

- 统一 resize 到固定大小

### 15.3 默认没有启用 raw audio augmentation

默认：

```python
raw_augment = 0
```

所以默认增强主要来自：

- `SpecAugment`

### 15.4 repeat padding 会引入伪重复模式

它能避免大量零填充，但也会把短周期重复拼接，可能改变部分频谱统计特性。

---

## 16. 用一句话总结本项目的数据预处理

可以把这套预处理总结成：

> 先根据 ICBHI 标注把整段录音切成单个呼吸周期，再统一到 8 秒长度，转换为 128-bin 的 Mel fbank 频谱图，训练阶段再做 SpecAugment 和尺寸归一化，最终形成固定大小的 2D 输入供模型训练。
